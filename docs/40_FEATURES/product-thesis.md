# Product Thesis

Git for Music is a music collaboration and demo versioning system.

It combines:

- Google Drive-style collaboration for live shared work
- Git-style history for branching, reverting, and comparing demos
- Studio-style audio processing where original audio is never destroyed

The product exists because musicians need a way to collaborate on demos without losing previous ideas, overwriting each other, or flattening the creative process into one destructive timeline.

The system should make experimentation feel safe.

A user should be able to:

- Upload or record a track
- Try an edit
- See who changed what
- Revert without fear
- Branch an older idea
- Compare versions
- Work offline
- Sync later
- Resolve musical conflicts without corrupting the demo
