# 2026-07-09

## Implementation Log

- Updated the demo version-tree layout so forks are centered around their
  parent node instead of pinning the parent to the leftmost child branch.
- Centered the rendered Tree tab content inside the scroll rail when there is
  spare horizontal space, so the graph reads as a centered object rather than
  a left-aligned strip.
- Added regression coverage for the centered fork layout in
  `src/app/pages/groups/demo/components/daw/version-graph-layout.test.ts`.
- Refreshed the branch-scoped tree-layout docs to remove the older
  left-to-right-by-depth language and describe the centered commit-graph
  behavior instead.
- Reworked the version nodes so they now show only the title and creator on the
  card, with the rest of the metadata moved into a click-open pop-up.
- Increased the node geometry and typography so the version graph is readable
  at a glance instead of using the previous compact node treatment.
- Derived the creator label from version history data (`actorUserId`) when the
  version snapshot itself does not carry a creator field, removing the
  `Unknown user` fallback for real commits.
- Tightened the node typography again so the title stays on one line and the
  author line remains visible beneath it.
- Increased the tree node footprint again and removed truncation behavior so
  the visible title/author text can live fully inside the node instead of
  relying on an ellipsis treatment.
- Removed the version-graph history lane and the client-side "Viewing history
  point" banner so the graph no longer exposes a selectable history-point
  surface.
