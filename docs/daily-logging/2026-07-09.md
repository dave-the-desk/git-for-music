# 2026-07-09

## Implementation Log

- Updated the demo DAW track lane so plugin chains are collapsed by default and can be expanded per track with a disclosure control.
- Reworked the recording card layout so the selected microphone state is presented in a tighter stacked arrangement with clearer ready/status messaging.
- Refined the microphone selector popup to better fit the selected-device workflow.
- Added interaction coverage for the collapsed-by-default plugin rack behavior and the expand-on-demand interaction.
- Restored the saved fake WAM plugin grant across every `demo3` demo instance so the module endpoint resolves in whichever `demo3` page is open.
- Added an automatic retry pass for plugin module preloading so a stale `Plugin chain error` banner clears once the module becomes available again.
- Added cache-busting query strings and no-store headers to the plugin module route so a previously cached module failure does not linger after access is restored.
- Removed the `sandbox` CSP from the plugin module proxy responses so dynamic `import()` keeps the module same-origin and can link the uploaded WAM body.
- Added a focused header helper test to lock in the import-safe plugin module response contract.
- Switched the WAM loader to fetch remote modules and import them from a generated `data:` URL so Next/Webpack no longer hijacks plugin module loading in the browser.
- Added shared commit-path and delete-track console logs in the demo DAW client to pinpoint unexpected track mutations during interaction tests.
- Added server-side support for `PLUGIN_ADDED`, `PLUGIN_REMOVED`, `PLUGIN_REORDERED`, `PLUGIN_PARAM_SET`, `PLUGIN_BYPASS_SET`, and `PLUGIN_STATE_SET` so plugin edits commit without a 500 and replay through snapshot rebuilds.
- Added a delete action to the account plugin library so owners can remove plugins from their library with confirmation.
- Added a separate plain `.js` WAM sample in the repo so the app can load a simpler plugin module path for validation.
- Switched the browser-side WAM loader to fetch plugin modules into blob URLs instead of data URLs so dynamic imports keep a same-origin module identity in the browser.
- Expanded the WAM loader and demo DAW console logs to stringify payloads and include the fetched module response metadata so import failures are easier to pinpoint.
- Moved the uploaded-track branch snapshot to the post-upload state so the version tree now branches from the node where the audio clip actually landed instead of the pre-upload empty track state.
- Changed recording saves to resolve the branch source version with the same freshest-committed logic used by uploads so a second armed-track recording branches from the latest committed version.
- Added a recording-save console log that prints the selected version, live active version, freshest committed version, and armed track target before upload so stale branch selection is easier to diagnose.
- Extended the demo DAW interaction test to arm the second track, record into it, and verify the resulting version still contains both tracks.
- Updated the feature docs to describe the freshest-committed source-version resolver for upload and recording workflows, and converted the old duplicate-track note into historical context.
