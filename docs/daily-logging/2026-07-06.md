# 2026-07-06

## Plugin Emit Helpers

- Added plugin command helpers to `AudioEditingEngine` for `PLUGIN_ADDED`, `PLUGIN_REMOVED`, `PLUGIN_REORDERED`, `PLUGIN_PARAM_SET`, `PLUGIN_BYPASS_SET`, and `PLUGIN_STATE_SET`.
- Wired `DemoDawClient` to resolve plugin operations to the owning track and expose thin async wrappers that commit the new plugin requests through the existing sync path.
- Kept the plugin helpers aligned with the existing edit pattern so commit requests still flow through `commitEditingOperation` / `commitOperation` with the same idempotency and target-resolution behavior as segment edits.

## WAM Host

- Added `src/app/lib/daw/engine/wam-host.ts` to load plugin ES modules by `pluginKey` and `version`, cache them by key/version, and bootstrap a per-`AudioContext` WAM runtime with `WamEnv`/`WamGroup` bookkeeping.
- Added `createInstance`, `applyParams`, and `applyState` helpers so plugin definitions can be promoted to live audio nodes and then configured from serialized state.
- Added a fail-fast main-thread guard so WAM host operations cannot run inside an `AudioWorklet` render context.
- Added `createWamPlaybackPluginGraphFactory` to build ordered insert chains from preloaded plugin state, skipping bypassed instances and returning the input node unchanged for empty chains.
- Added focused Vitest coverage for module caching, per-context bootstrap reuse, and parameter/state application.
- Added focused Vitest coverage for ordered chain building, bypass handling, and empty-chain passthrough.

## Playback Wiring

- Passed the WAM plugin graph factory into `AudioPlaybackEngine` from `DemoDawClient` so the running playback engine can resolve the current track plugin chain.
- Kept the factory stable with a ref-backed track lookup so the engine keeps seeing the latest plugin state without recreating the playback engine on every render.
- Added an interaction-test assertion that the demo client constructs the playback engine with a `pluginGraphFactory` option.

## Playback Live Updates

- Extended `AudioPlaybackEngine` with `rebuildTrackPluginChain(trackVersionId)` so chain changes can tear down and rebuild one track bus without restarting playback globally.
- Added live plugin parameter updates through `setPluginParam(trackVersionId, instanceId, paramId, value)` so knob changes can hit the active `WamNode` directly.
- Added `setPluginBypass(...)` as the chain-change trigger that forces a per-track rebuild when a bypass state changes.
- Updated the demo client plugin edit wrappers to route add/remove/reorder/param/bypass changes through the playback engine in addition to the existing commit path.
- Added focused Vitest coverage for live param updates, explicit rebuilds, bypass-triggered rebuilds, and param reapplication on `setProject()`.
- Refined `setProject()` so it diffs plugin chain structure separately from plugin params/state, reuses the live graph for param-only changes, and only rebuilds when the chain shape changes.

## Playback Disposal

- Updated `AudioPlaybackEngine.dispose()` to tear down each live track bus before clearing engine state so WAM nodes are disconnected and destroyed cleanly on shutdown.
- Kept track removal on the same teardown path so removing a bus also disposes its live plugin graph instead of just dropping the map entry.
- Added regression coverage for cleanup on track removal and on engine disposal.

## Playback Tests

- Added explicit `setProject()` coverage for the param-only path so plugin param changes reapply to the live graph without rebuilding it.
- Added explicit `setProject()` coverage for the chain-shape path so adding plugins forces a teardown and rebuild.
