# 2026-07-08

## User plugin uploads, personal library, per-demo sharing

Implemented the core plugin-library and sharing plumbing on top of the existing WAM insert v1.

### Work done

- Extended the Prisma schema with owner/visibility/module source fields on `PluginMetadata`, plus a new `PluginGrant` model and plugin enums.
- Added a Prisma migration for the new plugin tables/columns and generated the client.
- Added plugin storage-key builders alongside the existing track/audio builders.
- Built a new `packages/server/app/lib/plugins/` service for upload target creation, upload completion, library listing, per-demo grants, module access checks, and owner-only update/delete flows.
- Added plugin API routes for library listing, signed upload, completion, module proxy, and demo grant/revoke.
- Wired bootstrap plugin definitions to the demo-scoped availability query and included the descriptor URL / owner / visibility metadata.
- Updated `DemoDawClient.tsx` to preload WAM modules from the bootstrap `descriptorUrl` before rebuilding the playback graph.
- Updated the demo interaction fixtures to carry the new plugin metadata shape.
- Added a top-right header link to `/account/plugins` next to notifications, and added the private account plugins page route so the link resolves.
- Added drag-and-drop upload support on `/account/plugins`, including the sign-upload -> PUT -> complete-upload flow and a test for dropping a `.mjs` bundle.
- Added a generic `/account` homepage and moved the header entry point there; the bottom button on that page links to `/account/plugins`.
- Removed the explanatory subtext from `/account/plugins` so the page only shows the upload and library controls.
- Added [`docs/using-the-app.md`](../using-the-app.md) with a short end-to-end guide for sign-up, account access, plugin upload, and demo usage.
- Added [`docs/fake-wam-plugin.mjs`](../fake-wam-plugin.mjs), a small delay-effect WAM sample you can upload to `/account/plugins` and test in the DAW.
- Added a targeted test for the fake WAM sample module so its exported factory stays valid.
- Updated the DAW bootstrap plugin list so the current user's private uploads now appear in the Plugins tab, not just public or granted plugins.
- Added a regression test that proves a private owner plugin is visible in the DAW Plugins tab.
- Preloaded plugin modules before committing a new plugin insert so freshly added uploads do not race the playback graph rebuild.
- Restyled the occupied track insert-chain card to use a denser label/actions layout so plugin rows read cleanly when a track has inserts.
- Simplified the occupied plugin card further so the plugin name is the primary label, the track rack hides the bypass control once bypassed, and the rack keeps the state visible.

## DAW MP3 export buttons

Added MP3 export actions to the demo DAW footer so users can download either the current mix or one MP3 per track.

### Work done

- Added a browser-side export engine in `src/app/lib/daw/engine/export-engine.ts` that fetches the track audio, rebuilds the current segment layout, applies fades/gain/pan, and encodes real MP3 blobs with `lamejs`.
- Added a small module declaration for `lamejs` so TypeScript can typecheck the encoder import.
- Wired `DemoDawClient.tsx` to show two footer buttons: one for the full-project MP3 and one for individual track MP3 downloads.
- Kept the export flow aligned with the current DAW state by using the selected version, current segment layout overrides, tempo, and per-track gain/mute state.
- Added interaction coverage for the new footer buttons and a render helper regression test that exercises the export mix logic.
