# 2026-07-11

## Implementation Log

- Collapsed the docked version-history rail into a single predictable scroll
  path by removing the extra outer scroll wrappers around the tree.
- Added a rail-height-driven reset signal so the version-history tree resets
  again after the docked rail gets its measured height, not just on the first
  mount.
- Updated the tree and docked-rail interaction tests to cover the new reset
  signal and confirm the rail height feeds through to the tree props.
- Added zoom controls to the demo Version history rail header so users can zoom
  in, zoom out, and reset from the top-right control cluster.
- Kept the zoom state local to the history tree and applied it as a graph
  scale, so the version graph layout stays unchanged while the viewport zooms.
- Added interaction coverage for the zoom controls from the rail header.
- Moved the zoom controls into the top-right of the Version history rail, removed
  the shortcut hint text, and let zoom-out continue below the old 75% floor.
- Reworked the version-history graph layout so sibling branches share the same
  row, spread horizontally, and keep their parent centered above the full
  sibling span.
- Reworked the version-history palette so the active branch renders in blue,
  the active checkout node uses a lighter blue, off-branch nodes use a lighter
  yellow, and off-branch heads use a darker yellow.
- Hydrated version-history actor labels from resolved user display names in the
  snapshot state so the tree shows account names instead of raw actor IDs or
  unknown-user fallbacks.
- Persisted resolved creator names on version nodes so non-root entries render
  the creator's display name instead of falling back to `Unknown user`.
- Changed the root version-history node summary to read `[demo name] created`
  instead of the generic `Initial version` text.
- Stamped creator display names into every version-creation payload and
  propagated them through the snapshot and reducer path so newly created nodes
  render the author's name on first paint instead of waiting for later
  hydration.
- Switched the inspector measurement for the version-history rail to a
  layout-phase update so the rail mounts at its final height instead of
  flashing tall and then shrinking a frame later.
- Skipped the version-history rail auto-scroll on the initial mount so the tree
  opens at the top instead of jumping into view at the start.
- Removed the version-history rail auto-scroll path entirely so the page no
  longer jumps to the tree on load or when the tree tab becomes active.
- Forced the version-history scroll container back to `scrollTop = 0` on mount
  so the tree opens at the top even if the browser restores a nested scroll
  position.
- Removed the redundant rename button from the version-details popup and made
  the title itself open inline rename mode.
- Hid the "Go back to this version" action when the inspected node is already
  the selected checkout version.
- Removed the version ID and parent metadata rows from the version-details popup.
- Derived the branch field from the original branch source node so it stays in
  sync when that source node is renamed.
- Made the Operation field fall back to the matching history entry when the
  version snapshot does not carry an `operationSeq`.
- Removed the Operation field from the version-details popup entirely.
